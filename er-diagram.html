HotChix Restaurant Management System - ER Diagram

Database Schema Description

Customer

  • C_id (Primary Key): Unique identifier for each customer
  • C_name: Customer's full name
  • C_email: Customer's email address (unique)
  • C_contact: Customer's phone number
  • C_password: Hashed password for customer login
  • created_at: Timestamp when the customer account was created

Staff (Managers)

  • S_id (Primary Key): Unique identifier for each staff member
  • S_name: Staff member's full name
  • S_email: Staff member's email address (unique)
  • S_contact: Staff member's phone number
  • S_role: Role of the staff member (manager, admin, receptionist)
  • S_password: Hashed password for staff login
  • created_at: Timestamp when the staff account was created

Waiter

  • W_id (Primary Key): Unique identifier for each waiter
  • W_name: Waiter's full name
  • W_email: Waiter's email address (unique)
  • W_contact: Waiter's phone number
  • W_password: Hashed password for waiter login
  • W_experience: Years of experience
  • W_shift: Work shift (morning, evening, night)
  • created_at: Timestamp when the waiter account was created

Chef

  • CH_id (Primary Key): Unique identifier for each chef
  • CH_name: Chef's full name
  • CH_email: Chef's email address (unique)
  • CH_contact: Chef's phone number
  • CH_speciality: Chef's cuisine specialization
  • CH_password: Hashed password for chef login
  • CH_experience: Years of experience
  • CH_shift: Work shift (morning, evening, night)
  • created_at: Timestamp when the chef account was created

MenuCategory

  • category_id (Primary Key): Unique identifier for each menu category
  • category_name: Name of the category
  • description: Description of the category

MenuItem

  • item_id (Primary Key): Unique identifier for each menu item
  • category_id (Foreign Key): Reference to MenuCategory
  • item_name: Name of the menu item
  • description: Description of the menu item
  • price: Price of the menu item
  • is_veg: Boolean indicating if the item is vegetarian
  • is_available: Boolean indicating if the item is currently available
  • image_url: URL to the image of the menu item
  • chef_id (Foreign Key): Reference to Chef who specializes in this item

RestaurantTable

  • table_id (Primary Key): Unique identifier for each table
  • table_number: Table number in the restaurant
  • capacity: Number of people the table can accommodate
  • is_available: Boolean indicating if the table is currently available

Order

  • Order_Id (Primary Key): Unique identifier for each order
  • C_id (Foreign Key): Reference to Customer
  • W_id (Foreign Key): Reference to Waiter
  • S_id (Foreign Key): Reference to Staff
  • CH_id (Foreign Key): Reference to Chef
  • table_id (Foreign Key): Reference to RestaurantTable
  • Order_Date_Time: Timestamp when the order was placed
  • Order_Amount: Total amount of the order
  • Order_Status: Status of the order (Pending, Confirmed, Preparing, Ready, Served, Paid, Cancelled)
  • special_instructions: Any special instructions for the order
  • payment_method: Method of payment (Cash, Credit Card, Debit Card, Online)
  • payment_status: Status of payment (Pending, Completed, Failed)

OrderItem

  • order_item_id (Primary Key): Unique identifier for each order item
  • Order_Id (Foreign Key): Reference to Order
  • item_id (Foreign Key): Reference to MenuItem
  • quantity: Quantity of the menu item ordered
  • item_price: Price of the menu item at the time of ordering
  • special_instructions: Any special instructions for the specific item

Reservation

  • reservation_id (Primary Key): Unique identifier for each reservation
  • C_id (Foreign Key): Reference to Customer
  • table_id (Foreign Key): Reference to RestaurantTable
  • reservation_date: Date of the reservation
  • reservation_time: Time of the reservation
  • num_guests: Number of guests
  • status: Status of the reservation (Pending, Confirmed, Cancelled)
  • special_request: Any special requests for the reservation
  • created_at: Timestamp when the reservation was created

Entity Relationships

  • Customer to Order: One-to-Many (One customer can place many orders)
  • Customer to Reservation: One-to-Many (One customer can make many reservations)
  • Waiter to Order: One-to-Many (One waiter can serve many orders)
  • Staff to Order: One-to-Many (One staff member can manage many orders)
  • Chef to Order: One-to-Many (One chef can prepare many orders)
  • Chef to MenuItem: One-to-Many (One chef can specialize in many menu items)
  • MenuCategory to MenuItem: One-to-Many (One category can have many menu items)
  • Order to OrderItem: One-to-Many (One order can have many order items)
  • MenuItem to OrderItem: One-to-Many (One menu item can be in many order items)
  • RestaurantTable to Order: One-to-Many (One table can have many orders)
  • RestaurantTable to Reservation: One-to-Many (One table can have many reservations)
